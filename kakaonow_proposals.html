카카오 나우 섹션 UX 개선 제안

"배너 수를 2 → 4 → 6으로 늘리면 노출 기회도 늘어난다"는 조직 내 직관은 사용자 행동과 반대 방향입니다. 이 문서는 현재 2×3 그리드의 구조적 문제를 진단하고, 정치적 현실(6개 슬롯 보존 필요)까지 고려한 4가지 시안을 비교합니다.

문제 진단

01 · SPACE GREED
2×3 그리드(카드 aspect 175:204)가 뷰포트 1 스크린의 약 45%를 점유. 더보기탭의 "빠른 진입·탐색" 목적과 충돌.
02 · VISUAL NOISE
각 배너가 배경색·타이포·일러스트 크기를 자유롭게 사용 → 6개가 한 화면에서 경쟁. 개별 가독성은 물론 섹션 전체가 "광고존"으로 인지되어 건너뛰게 됨.
03 · CTR DILUTION
6개 선택지 중 1개 클릭. 이론상 CTR 상한이 1/6로 분산. "가끔 특정 소재에서 튄다"는 것은 포맷이 아니라 콘텐츠 자체가 드라이버임을 시사.
04 · EGO-DRIVEN IA
사용자 가치가 아닌 내부 협상으로 슬롯 수가 정해짐. 지표 하락 추세는 이 정책의 외부 검증 실패 신호.

상태

6개 슬롯 동등 노출. 세로 공간 과점유, 소재 디자인 비일관, 선택지 과다.

  • 세로 공간 과점유 (뷰포트 약 45%)
  • 배너 간 위계 없음 → 모두 무시되는 banner blindness
  • CTR 분산 / 지표 하락 추세
시안 A 01

싱글 히어로 + 자동 롤링

시안 A · 히어로 롤링
카카오 나우
UX 근거 · Peak-End / Attention

한 순간 한 개의 배너만 집중 노출 → 주의 경제(attention economy) 준수. 자동 롤링 5초 + 스와이프로 6개 모두 노출 기회 보장. 원래 포맷으로의 회귀 + 현대화.

  • 세로 공간 약 70% 축소 (1 영역만 표시)
  • 각 배너가 최대 주목도 확보 → 소재 품질이 CTR에 직결
  • 한 스크린에 동시 노출은 1개 → 일부 배너는 롤링 전 이탈
  • "6개 슬롯 모두 보존, 순차 순환" 프레이밍으로 내부 설득 가능
시안 B · 추천 02

가로 스와이프 피크 캐러셀

시안 B · 피크 스트립
카카오 나우
UX 근거 · Progressive Disclosure

첫 화면에서 2.5개 카드 peek으로 "더 있음"을 암시 → 스와이프 어포던스 확보. 슬롯 6개 보존하면서 세로 공간 1/3로 축소. Netflix·Toss·Spotify 등 친숙한 row 패턴.

  • 세로 공간 약 65% 축소, 6개 즉시 스캔 가능
  • 사용자가 능동적으로 탐색 → banner blindness 완화
  • 배너 소재 품질 차이가 노출 순서로 조정 가능
  • 후순위 카드는 스와이프 없으면 노출 0
  • "6개 전부 보임 + 공간 절약" 양쪽 요구 동시 충족

UX 근거 · Design System Enforcement

레이아웃은 유지하되 카드 내부를 컴포넌트화: 파스텔 배경 + 원형 아이콘 + 1줄 제목 + 1줄 서브. 배경 이미지에 텍스트 박는 자유 금지. 최소 개입으로 가독성·일관성 확보.

  • 최소 변경 → 내부 이해관계자 저항 최소
  • 디자인 토큰 강제로 "배너 가지각색" 문제 근본 해결
  • 카드 세로를 175:204 → 160:128로 줄여 공간도 20%↓
  • 공간 절감 폭은 B·C보다 작음, 선택지 과다 문제는 남음
  • "슬롯·레이아웃 모두 유지, 품질만 개선" 가장 정치적 안전

권장 전략 · 단계적 접근

단일 정답은 없음. 4개 시안은 공간 절감 강도와 정치적 수용성의 트레이드오프 축에 놓여 있습니다. 지표 하락 추세와 "소재에 따라 튀는" 패턴은 포맷이 아니라 콘텐츠 품질이 핵심임을 시사합니다. 따라서 포맷 개선만으로 해결되지 않으며, 소재 품질 게이트 + 개인화 슬롯이 중장기 해답입니다.

단기적으로는 시안 B (피크 캐러셀)가 "6 슬롯 보존 + 공간 절감 + 탐색 어포던스"의 세 조건을 동시 만족하므로 내부 설득 비용 대비 가장 효율적입니다.

단계별 로드맵 제안

  1. STEP 1 · 긴급 (2주) — 시안 B 적용 + 소재 디자인 가이드 1.0 배포 (배경 이미지 위 텍스트 금지, 타이포 토큰 강제)
  2. STEP 2 · 측정 (4주) — 스와이프율·포지션별 CTR A/B, 슬롯 6개 필요성 데이터 검증
  3. STEP 3 · 재설계 (8주) — 슬롯 수를 데이터 기반으로 재협상, 피처드 슬롯(시안 C 상단) 1개를 개인화 추천으로 전환
  4. STEP 4 · 제도화 — "배너 추가"는 소재 품질 게이트 통과 시에만 가능하도록 내부 프로세스 신설
