문제 진단

현재 2×3 그리드는 단순히 "공간을 많이 차지한다"의 문제가 아닙니다. 구조·시각·지표·의사결정의 네 축에서 동시에 무너지고 있습니다.

01 · Space Greed

뷰포트 절반 가까이 점유

카드 aspect 175:204 × 6개가 1 스크린의 약 45%를 차지. 더보기탭의 "빠른 진입·탐색" 목적과 정면 충돌.

02 · Visual Noise

배너가 서로 경쟁

각 배너가 배경색·타이포·일러스트 크기를 자유롭게 사용. 6개가 한 화면에서 경쟁하면 섹션 전체가 "광고존"으로 인지되어 건너뛰게 됨.

03 · CTR Dilution

선택지 과다의 역설

6개 중 1개 클릭. 이론상 CTR 상한이 1/6로 분산. "특정 소재에서 튄다"는 현상은 포맷이 아니라 콘텐츠 자체가 드라이버임을 시사.

04 · Ego-driven IA

사용자 가치 부재

사용자 가치가 아닌 내부 협상으로 슬롯 수가 결정됨. 지표 하락 추세는 이 정책의 외부 검증 실패 신호.

시안 비교

각 시안은 실제 더보기탭 전체 흐름에 적용된 상태로 아래에 라이브 렌더링되어 있습니다. 프레임을 스크롤해 섹션 전후의 컨텍스트까지 확인하세요. 섬네일 우상단의 ↗ 버튼으로 새 탭에서 전체 화면으로도 열 수 있습니다.

AS-IS 00

현재 — 2×3 풀 그리드

열기
상태

6개 슬롯 동등 노출. 세로 공간 과점유, 소재 디자인 비일관, 선택지 과다.

  • 세로 공간 과점유 (뷰포트 약 45%)
  • 배너 간 위계 없음 → banner blindness
  • CTR 분산 / 지표 하락 추세
시안 A 01

싱글 히어로 + 자동 롤링

열기
Peak-End / Attention

한 순간 한 개의 배너만 집중 노출. 자동 롤링 5초 + 점 클릭으로 6개 모두 노출 기회 보장. 원래 포맷으로의 회귀 + 현대화.

  • 세로 공간 약 70% 축소
  • 각 배너 최대 주목도 확보
  • 동시 노출 1개 → 일부 배너는 이탈 전 노출 0
  • "6개 슬롯 모두 보존, 순차 순환" 프레이밍
시안 B · 추천 02

가로 스와이프 피크 캐러셀

열기
Progressive Disclosure

첫 화면에서 2.5개 카드 peek으로 "더 있음"을 암시 → 스와이프 어포던스 확보. 세로 공간 1/3로 축소. Netflix·Toss·Spotify 등 친숙한 row 패턴.

  • 세로 공간 약 65% 축소
  • 6개 즉시 스캔 + 능동적 탐색
  • 노출 순서로 소재 품질 조정 가능
  • 후순위 카드는 스와이프 없으면 노출 0
  • "6 슬롯 + 공간 절약" 양쪽 요구 동시 충족
시안 C 03

피처드 + 리스트 (1+5)

열기
Visual Hierarchy

1 히어로는 주목·감성, 5 리스트는 텍스트 일관 포맷으로 스캔성 확보. YouTube 홈·App Store Today의 패턴. 리스트가 "제각각 디자인" 문제를 구조적으로 차단.

  • 위계 확보 → 첫 소재 집중 노출
  • 리스트 포맷 강제 → 디자인 일관성
  • 5개 리스트는 비주얼 임팩트 손실
  • "6 슬롯 보존 + 위계" 타협안
시안 D 04

2×3 유지 · 디자인 토큰 통일

열기
Design System Enforcement

레이아웃은 유지하되 카드 내부를 컴포넌트화: 파스텔 배경 + 원형 아이콘 + 1줄 제목 + 1줄 서브. 최소 개입으로 가독성·일관성 확보.

  • 최소 변경 → 이해관계자 저항 최소
  • 디자인 토큰 강제 → 품질 통제
  • 카드 비율 축소(aspect 168:132)로 공간 20%↓
  • 선택지 과다 문제는 남음
  • "슬롯·레이아웃 유지, 품질만 개선" 최안전

권장 전략

단일 정답은 없습니다. 4개 시안은 공간 절감 강도와 정치적 수용성의 트레이드오프 축에 놓여 있습니다. 지표 하락 추세와 "소재에 따라 튀는" 패턴은 포맷이 아니라 콘텐츠 품질이 핵심임을 시사하므로, 포맷 개선은 필요조건이지 충분조건이 아닙니다.

DO

  • 시안 B(피크 캐러셀)를 단기 적용 — "6 슬롯 보존 + 공간 절감 + 탐색 어포던스" 3조건 동시 충족
  • 소재 디자인 가이드 1.0 제정 — 배경 이미지 위 텍스트 금지, 타이포·여백 토큰 강제
  • 포지션별 CTR·스와이프율 A/B — 슬롯 6개의 필요성을 데이터로 재검증
  • 피처드 슬롯 1개를 개인화 추천으로 전환 — 사용자 행동 기반 소재 매칭

DON'T

  • 슬롯을 7, 8개로 계속 늘리는 것 — 지표 하락 가속만 초래
  • 배너 추가를 사전 데이터 검증 없이 승인하는 프로세스
  • 배경 이미지 위에 본문 텍스트를 새겨 넣는 배너 소재
  • "6개 전부 동등" 레이아웃 — 위계 없이 모두가 광고로 인지됨
STEP 1 · 긴급 · 2주

시안 B 적용 + 소재 디자인 가이드 1.0 배포. 원본 지면 대비 세로 공간 65% 절감, 슬롯 수는 유지.

STEP 2 · 측정 · 4주

스와이프율·포지션별 CTR A/B. 6개 슬롯의 기여도를 포지션별로 수치화하여 필요성 재검증.

STEP 3 · 재설계 · 8주

슬롯 수를 데이터 기반으로 재협상. 피처드 슬롯 1개를 개인화 추천으로 전환 (시안 C 상단 포지션 활용).

STEP 4 · 제도화

배너 추가는 소재 품질 게이트 통과 시에만 가능하도록 내부 프로세스 신설.
